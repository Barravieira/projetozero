Este projeto é um SaaS com suporte a múltiplos tenants (multi-tenant), criado com React + Vite + Firebase. Ele utiliza Mantine versão 8.0.0 como biblioteca de interface. É obrigatório ler e seguir o arquivo .cursor/rules/general.mdc, que contém todas as diretrizes técnicas do projeto.
Neste modelo, cada usuário pertence a uma organização ou empresa (um tenant). Após o cadastro, o usuário deve ser vinculado a um tenant no Firestore. Todos os dados do sistema devem conter o campo tenantId, que indica a qual organização aquele dado pertence. Nenhum dado deve ser acessível por usuários de outros tenants.
As regras de segurança do Firestore devem garantir que os usuários só possam acessar dados com tenantId correspondente ao seu. Os dados podem ser organizados em subcoleções do tipo /tenants/{tenantId}/colecao ou em coleções comuns com o campo tenantId.
A interface deve ser feita exclusivamente com Mantine 8.0.0. Use @mantine/core, @mantine/form, @mantine/notifications e @mantine/modals. Não use bibliotecas externas de UI nem HTML puro.
O roteamento deve seguir a estrutura:
/app/auth para login, cadastro e onboarding
/app para rotas privadas e o painel principal
Pode haver uma tela inicial para criar ou selecionar o tenant após o cadastro
Os arquivos devem seguir a estrutura do projeto: páginas em src/pages/, serviços em src/services/, e organização clara dos dados. O tenantId pode ser controlado via Context API do React, sem bibliotecas externas de estado.
